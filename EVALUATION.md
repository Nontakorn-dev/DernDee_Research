# Evaluation Protocol

This repo evaluates TinyTCN compression at **100 Hz** with **0.5 s causal windows**. The paper should not report 50 Hz, 1 s windows, or ESP32 numbers unless those experiments are explicitly run and measured.

## Subject-wise split

The fixed split is `shared/splits/subject_split.csv`:

- 24 train subjects
- 5 validation subjects
- 6 test subjects
- seed 42 when regenerated by `shared/scripts/make_split.py`

```bash
python shared/scripts/make_split.py
```

## FP32 model training (shared protocol)

All models (`tinytcn`, `cnn1d`, `lstm_gru`, `tcn`, `transformer`) share:

```bash
bash experiments/scripts/train_all_colab.sh \
  --config shared/configs/train_fair_comparison.json
```

Single model example:

```bash
bash experiments/tinytcn/scripts/train.sh \
  --config shared/configs/train_fair_comparison.json
```

Default training settings (from `shared/configs/train_fair_comparison.json`):

- channels: bilateral, 12 IMU inputs
- source rate: 200 Hz
- target rate: 100 Hz
- decimation: 2
- window: 50 samples (0.5 s)
- train stride: 5 samples (0.05 s)
- validation/test stride: 1 sample
- epochs: 50 (checkpoint saved at best validation macro F1)
- batch size: 512
- optimizer: Adam, lr 1e-3
- model hidden width: 32

Outputs go to `experiments/tinytcn/runs/fp32_100hz/`.

## Metrics

The primary paper metrics are:

- overall accuracy
- macro F1
- phase-specific F1 for LR, LS, PSw, Sw
- phase-specific accuracy, computed as diagonal / support from the confusion matrix
- model size in KB

Use structured metrics where possible:

```bash
python shared/eval_checkpoint.py \
  --checkpoint experiments/tinytcn/runs/fp32_100hz/best_model.pt \
  --out experiments/tinytcn/runs/fp32_100hz/test_metrics.json
```

## Compression

PyTorch-first compression evaluation is the accuracy source of truth before ESP32 deployment:

```bash
bash experiments/compression/scripts/run_compression.sh
```

This evaluates:

- FP32
- INT8 weight quantize-dequantize proxy
- INT4 weight quantize-dequantize simulation
- Prune50 structured hidden-channel pruning with fine-tuning
- INT8+Prune50

Results are written under:

- `experiments/compression/runs/<CONFIG>/metrics.json`
- `experiments/compression/results/metrics.json`
- `experiments/compression/overrides.json`
- `experiments/compression/manifest.json`
